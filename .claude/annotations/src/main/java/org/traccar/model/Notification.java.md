# Notification.java

**Role:** A notification subscription rule. `@StorageName("tc_notifications")` ‚Üí `tc_notifications`. Defines which event type triggers a notification, on which channel(s), and optionally restricted to a calendar window.
**Fits in:** Extends `ExtendedModel` (has attributes). Implements `Schedulable`. `NotificationManager` loads all notifications linked to a device's users/groups, matches incoming `Event.type` against `Notification.type`, then fires the listed notificators.
**Read next:** [[Event.java]] (triggers), [[Calendar.java]] (time gating), [[Schedulable.java]]

## Public API

### DB-mapped fields
- `description String` (line 29) ‚Äî display label.
- `calendarId long` (line 38) ‚Äî FK to `tc_calendars`; 0 = always active.
- `always boolean` (line 50) ‚Äî if true, fire even if the matched event is generated by the same user who owns the notification.
- `type String` (line 60) ‚Äî matches `Event.TYPE_*`; `"allEvents"` = subscribe to everything.
- `commandId long` (line 70) ‚Äî FK to `tc_commands`; used by the `command` notificator channel.
- `notificators String` (line 80) ‚Äî comma-separated delivery channel names: `"mail"`, `"sms"`, `"firebase"`, `"telegram"`, `"web"`, `"command"`, etc.

### Not DB-mapped
- `getNotificatorsTypes()` (line 93) ‚Äî parses `notificators` CSV into a `Set<String>`.

## Gotchas / non-obvious

- `type = "allEvents"` is the wildcard; `NotificationManager` checks `type.equals(Event.ALL_EVENTS) || type.equals(event.getType())`.
- `commandId` is only meaningful when `notificators` contains `"command"`.
